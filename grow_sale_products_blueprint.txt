1. PROJECT ARCHITECTURE & DESIGN SCHEMA
--------------------------------------------------------------------------------
To honor the "antigravity" concept of Python (simplicity, elegance, and rapid 
development), the system is divided into two operational core engines:
  A. The Showcase & Lead Engine (Services, Materials, Quote Intake)
  B. The Authority Blog Engine (Content Management, Client Education)

[Database Schema Mapping (MySQL)]

Table 1: blog_posts
  - id (INT, Primary Key, Auto Increment)
  - title (VARCHAR 200)
  - slug (VARCHAR 200, Unique)
  - content (LONGTEXT / Rich Text)
  - featured_image (VARCHAR 255 - File path/URL)
  - status (VARCHAR 20 - Choice: Draft, Published)
  - created_at (DATETIME)
  - updated_at (DATETIME)

Table 2: service_categories
  - id (INT, Primary Key)
  - name (VARCHAR 100) -> e.g., "Garment & Apparel", "Industrial Screen Printing"
  - slug (VARCHAR 100)
  - description (TEXT)
  - resolution_dpi (VARCHAR 50) -> e.g., "600 to 1200 DPI"

Table 3: quotes
  - id (INT, Primary Key)
  - client_name (VARCHAR 150)
  - company_email (VARCHAR 150)
  - phone (VARCHAR 20)
  - material_choice (VARCHAR 50) -> e.g., Fabric, Silk, Plastics, Cardboard
  - finish_type (VARCHAR 30) -> Glossy, Matte, Textured
  - design_file (VARCHAR 255 - Media storage route)
  - message (TEXT)
  - submitted_at (DATETIME)

2. FRONT-END DESIGN & ANIMATION TOKENS
--------------------------------------------------------------------------------
The interface balances a heavy industrial printing vibe with smooth, liquid
digital micro-interactions.

Color Palette:
  - Ink Charcoal:   #1A1D20 (Primary background elements, heavy titles)
  - Squeegee Cyan:  #00E5FF (Accents, active borders, digital CTAs)
  - Mesh Silver:    #F4F6F9 (Main light body backgrounds for stark contrast)
  - Textile Gold:   #FFB300 (Highlights: 26+ Years Badge, Premium Quotes)

Animation Framework:
  - GSAP (GreenSock): Handles the complex landing hero animation (simulating a 
    screen printing squeegee sweeping across the screen revealing text).
  - AOS (Animate on Scroll): Handles simple structural elements fading and 
    sliding upwards seamlessly as the client scrolls down.

3. STEP-BY-STEP IMPLEMENTATION PLAN
--------------------------------------------------------------------------------

PHASE 1: ENVIRONMENT SETUP & MYSQL INTEGRATION
  1. Initialize local project folder structure:
     $ mkdir grow_sale_site && cd grow_sale_site
  2. Create and run Python virtual environment:
     $ python -m venv venv
     $ source venv/bin/activate  # On Windows: venv\Scripts\activate
  3. Install core packages:
     $ pip install django mysqlclient django-crispy-forms pillow
  4. Access MySQL terminal or Workbench and create database:
     CREATE DATABASE grow_sale_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  5. Edit `settings.py` to point to the MySQL database:
     DATABASES = {
         'default': {
             'ENGINE': 'django.db.backends.mysql',
             'NAME': 'grow_sale_db',
             'USER': 'your_mysql_user',
             'PASSWORD': 'your_secure_password',
             'HOST': 'localhost',
             'PORT': '3306',
         }
     }

PHASE 2: CORE DJANGO APPS & MODELS CREATION
  1. Create apps for 'blog' and 'services':
     $ python manage.py startapp blog
     $ python manage.py startapp services
  2. Define schemas in `blog/models.py` and `services/models.py` as outlined 
     in the structural mapping above.
  3. Run data migrations:
     $ python manage.py makemigrations
     $ python manage.py migrate
  4. Register models in `admin.py` with modified `prepopulated_fields` to 
     automatically create slugs from titles.

PHASE 3: COMPOSING BASE LAYOUTS & REUSABLE FRONTERA
  1. Create a global `templates/` directory containing a robust `base.html`.
  2. Inject external scripts via CDN securely inside `base.html`:
     - Tailwind CSS (<script src="https://cdn.tailwindcss.com"></script>)
     - AOS CSS & JS links for scroll handling.
     - GSAP library scripts.
  3. Formulate the dynamic navbar incorporating operational timings 
     (Mon-Sat: 10AM-8PM) and click-to-call link for "+91 99253 91567".

PHASE 4: DEVELOPING INTERACTIVE FRONT PAGES
  1. Hero Component: Use GSAP to animate a cyan line overlay sliding horizontally 
     across a deep charcoal container, wiping into view the message: 
     "26+ Years of Precision Screen Printing Excellence."
  2. Service Grid: Use dynamic template iteration `{% for service in services %}`. 
     Attach `data-aos="fade-up"` tokens to each card wrapper block so they float 
     up nicely during navigation.
  3. Blog Grid: Showcase industry authority tips (e.g., "Choosing Between 600 vs 
     1200 DPI for Silk Textiles").

PHASE 5: CUSTOM QUOTATION MANAGEMENT SYSTEM
  1. Construct a Django ModelForm matching the `quotes` database criteria.
  2. Setup static and media files collection parameters inside `settings.py` to 
     safely bucket incoming graphic mockups uploaded by cargo customers.
  3. Build view logics (`views.py`) validating post data payloads and flashing 
     smooth success modals using custom Tailwind toast notifications.

PHASE 6: AUDITING, PERFORMANCE & PRODUCTION POLISH
  1. Check query metrics via Django Debug Toolbar to ensure zero 'N+1' performance 
     leaks during database retrieval steps.
  2. Secure the framework: set `DEBUG = False` and update `ALLOWED_HOSTS`.
  3. Implement local caching strategies for static asset bundles.

--------------------------------------------------------------------------------
4. ANTIGRAVITY COMPLIANCE (MAINTAINING CLEAN ARCHITECTURE)
--------------------------------------------------------------------------------
- Avoid deep inheritance paths in models. Keep fields explicit.
- Use context processors for common details like phone numbers and operational 
  hours so changes in the Ahmedabad commercial listing sync instantly across 
  all footers, headers, and contacting sections without repeating template logic.
